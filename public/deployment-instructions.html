Website Deployment Instructions

Deploy to Vercel & Connect Domain

1 Create GitHub Account (if needed)

  1. Go to https://github.com
  2. Click "Sign up" in the top right corner
  3. Follow the prompts to create your account
  4. Verify your email address

2 Upload Your Code to GitHub

Option A: Using GitHub Desktop (Easiest)

  1. Download GitHub Desktop from https://desktop.github.com
  2. Install and sign in with your GitHub account
  3. Click "File" → "Add Local Repository"
  4. Browse to your /workspace folder
  5. Click "Create Repository" if prompted
  6. Name it: wavecrest-legacy-site
  7. Click "Publish repository" button
  8. Make sure "Keep this code private" is UNCHECKED (Vercel needs access)
  9. Click "Publish Repository"

Option B: Using GitHub Website

  1. Log in to GitHub.com
  2. Click the "+" icon in top right → "New repository"
  3. Name: wavecrest-legacy-site
  4. Leave as PUBLIC (Vercel needs access)
  5. DO NOT initialize with README
  6. Click "Create repository"
  7. On your computer, compress your workspace folder as a ZIP
  8. On the GitHub page, click "uploading an existing file"
  9. Drag and drop your files (NOT the zip, extract it first)
  10. Click "Commit changes"

3 Deploy to Vercel

  1. Go to https://vercel.com
  2. Click "Sign Up"
  3. Choose "Continue with GitHub"
  4. Authorize Vercel to access your GitHub account
  5. Once logged in, click "Add New..." → "Project"
  6. Find your wavecrest-legacy-site repository
  7. Click "Import"
  8. IMPORTANT: Vercel will auto-detect Next.js settings - just click "Deploy"
  9. Wait 2-3 minutes for deployment to complete
✓ Success! Your site is now live! Vercel will show you a URL like wavecrest-legacy-site.vercel.app

4 Connect Your Domain (wavecrestlegacy.com)

Part A: Add Domain in Vercel

  1. In Vercel, go to your project
  2. Click "Settings" tab
  3. Click "Domains" in the left sidebar
  4. Type: wavecrestlegacy.com
  5. Click "Add"
  6. Also add: www.wavecrestlegacy.com
  7. Click "Add"
Note: Vercel will show you DNS settings that look like this:
  • Type: A
  • Name: @ (or leave blank)
  • Value: 76.76.21.21
And:
  • Type: CNAME
  • Name: www
  • Value: cname.vercel-dns.com
Write these down!

Part B: Update DNS at IncAuthority.com

  1. Log in to incauthority.com
  2. Go to your domain management / DNS settings
  3. Find "DNS Records" or "Manage DNS" section
  4. Delete any existing A records or CNAME records for @ and www
  5. Add new A record:
    • Type: A
    • Host/Name: @ (or leave blank for root domain)
    • Value/Points to: 76.76.21.21
    • TTL: 3600 (or leave default)
  6. Add new CNAME record:
    • Type: CNAME
    • Host/Name: www
    • Value/Points to: cname.vercel-dns.com
    • TTL: 3600 (or leave default)
  7. Save your changes

5 Wait for DNS Propagation

  • DNS changes can take 10 minutes to 48 hours to propagate
  • Usually it's much faster (10-30 minutes)
  • Visit https://wavecrestlegacy.com to check
  • If it doesn't work right away, be patient and try again in 30 minutes
✓ You're Done! Your website will be live at wavecrestlegacy.com

Making Updates Later

When you need to make changes to your site:

  1. Make changes to your local files
  2. Push the changes to GitHub (using GitHub Desktop or website)
  3. Vercel automatically detects the changes and redeploys
  4. Your site updates in 2-3 minutes automatically!

⚠️ Important Notes

  • Repository must be PUBLIC for free Vercel deployment
  • Keep your GitHub credentials safe - you'll need them to make updates
  • Vercel is completely free for this type of site
  • SSL certificate is automatic - Vercel provides HTTPS for free

📞 Need Help?

If you run into issues:

  • GitHub Issues: Check GitHub's help documentation
  • Vercel Support: Vercel has excellent documentation at vercel.com/docs
  • DNS Issues: Contact IncAuthority support if DNS settings are confusing
